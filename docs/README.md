# docs/

Operational and architecture documentation. The top-level docs at the repo root
(`README.md`, `ARCHITECTURE.md`, `CONTEXT.md`, `ANALYSIS_LOG.md`, `TODO.md`,
`TUNING_NOTES.md`, `GO_LIVE_CHECKLIST.md`) cover the project's history,
strategy validation, and engineering log. The files in *this* directory cover
day-to-day operation of the running server.

| File | When to read |
|------|-------------|
| [`DEPLOYMENT.md`](DEPLOYMENT.md) | First-time setup on a new machine; reinstalling launchd |
| [`AUTO_EXECUTE.md`](AUTO_EXECUTE.md) | Before arming headless options auto-execution |
| [`RUNBOOK.md`](RUNBOOK.md) | Daily ops, troubleshooting, "the server isn't trading — why?" |

Project-wide guidance for Claude Code agents lives in
[`../CLAUDE.md`](../CLAUDE.md).
